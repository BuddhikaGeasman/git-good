## Auth Redirect Bug on Page Refresh (Production Only)

### The Issue
When refreshing the browser on the Analysis route (e.g. `/analysis/74?filterDays=30&period=day&selected=...`), the app incorrectly redirects the user to `/discovery` instead of staying on the current page. This only occurs on the **production build** hosted on Cloudflare — it does not reproduce on dev or stage builds hosted on AWS Amplify.
### Root Cause
Zustand's `persist` middleware restores state from `localStorage` via an **asynchronous microtask**, even though `localStorage` itself is synchronous. This creates a race condition between auth state hydration and TanStack Router initialization:

1. Page refreshes → SPA loads from scratch
2. Auth store is created with initial state (`loggedIn: false`)
3. Zustand schedules hydration from `localStorage` as a microtask
4. TanStack Router initializes and `authenticatedRoute.beforeLoad` fires **before** hydration completes
5. `useAuth.getState().loggedIn` is still `false` → redirects to `/auth/login`
6. Hydration finishes → `loggedIn` becomes `true`
7. Login route's `beforeLoad` sees `loggedIn: true` → redirects to `/discovery`

The end result: the user lands on `/discovery` instead of their original route.
### Why Only Production?
The race condition exists in all environments, but the outcome depends on how fast the JS bundle is delivered and executed:

| Environment | Hosting      | Behavior |
|------------|--------------|----------|
| Dev        | Vite dev server (local) | HMR + unoptimized modules add enough delay for hydration to complete first |
| Stage      | AWS Amplify  | Slightly higher latency to serve assets gives hydration time to flush |
| Production | Cloudflare   | Edge network with aggressive caching serves assets with near-zero latency — the router fires before the microtask queue processes hydration |

Same code, same build pipeline — Cloudflare is simply fast enough to expose the race condition.

### Fix
Await Zustand's persist hydration in all route guards before checking auth state:
```ts
const waitForAuthHydration = async () => {
  if (useAuth.persist.hasHydrated()) return;
  await new Promise<void>((resolve) => {
    const unsub = useAuth.persist.onFinishHydration(() => {
      unsub();
      resolve();
    });
  });
};

// In route guards:
beforeLoad: async () => {
  await waitForAuthHydration();
  if (!useAuth.getState().loggedIn) {
    throw redirect({ to: "/auth/login" });
  }
},
```
